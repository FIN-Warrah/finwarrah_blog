    版本控制（Git）

    1 一个简单的实例——上传此博客到 Github

    1.1 创建虚拟环境并安装 mkdocs

    conda create -n mkdocs
source activate mkdocs
conda install python
pip install mkdocs

    1.2 创建博客项目

    cd ~ Blog
mkdocs new myblog

    1.3 安装 material 主题

    pip install mkdocs-material

    1.4 修改博客配置

    cd myblog
vim mkdocs.yml

    修改 mkdocs.yml 内容如下：

    site_name: FIN-Warrah's Blog
theme:
  name: material

nav:
  - Git 与 GitHub:
    - 1 以上传本博客到 GitHub 为例: Git与GitHub/1_以上传本博客到GitHub为例.md

    其中，site-name 为站点名称，theme 为使用的主题；nav 为博客目录结构（博客左侧导航栏结构）

    注：此时 myblog 的结构为： 

    myblog
  - docs
    -  Git 与 GitHub
      - 1_以上传本博客到GitHub为例.md
    - index.md
  mkdocs.yml

    1.5 在 GitHub 创建仓库

    GitHub - New repository - 设置仓库名称（以 myblog 为例）

    1.6 使用 Git 对项目进行初始化及上传

    设置 .gitignore

    vim .gitignore

    内容为：

    site
docs

    Git 初始化与上传

    git init
git branch -m main （修改分支名称）
git add .
git commit -m "initialization"
git remote add origin [从GitHub仓库找到链接，形式为 https://github.com/用户名/仓库名.git]
git push -u origin main
    此时要求输入 GitHub 用户名和密码。

    可能无法直接使用密码登录，则在 GitHub - Settings - Developer settings - Personal access tokens 创建 token，并注意妥善保存（之后无法再看到这个 token 的内容了）

    重新执行 git push -u origin main，使用上述 token 代替密码，完成上传

    1.7 博客发布

    mkdocs gh-deploy

    至此，博客发布完成。

    1.8 所涉及的知识点

    Git 常用命令总结

    • .gitignore: 忽略哪些目录和文件
    • git init: Git 初始化
    • git branch -m main：将当前分支名称修改为 main
    • git add .: 将当前全部文件添加到 Git
    • git commit -m "备注内容": Git 内容提交
    • git push origin main: 上传到远程仓库
