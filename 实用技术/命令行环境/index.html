    1 作业控制（Job Control）

    1.1 信号（Signal）

    通过 Ctrl-C 终止程序的执行。具体来说，发送了一个 SIGINT (Signal Interrupt) 信号，告诉程序终止执行。 

    void@ubuntu:~$ sleep 20
^C

    通过 man signal 查看信号清单

     signal – simplified software signal facilities

LIBRARY
     Standard C Library (libc, -lc)

     sig_t

DESCRIPTION
     This signal() facility is a simplified interface to the more general
     sigaction(2) facility.

     Signals allow the manipulation of a process from outside its domain, as
     well as allowing the process to manipulate itself or copies of itself
     (children).  There are two general types of signals: those that cause
     termination of a process and those that do not.  Signals which cause
     termination of a program might result from an irrecoverable error or might
     be the result of a user at a terminal typing the `interrupt' character.
     Signals are used when a process is stopped because it wishes to access its
     control terminal while in the background (see tty(4)).  Signals are
     optionally generated when a process resumes after being stopped, when the
     status of child processes changes, or when input is ready at the control
     terminal.  Most signals result in the termination of the process receiving
     them, if no action is taken; some signals instead cause the process
     receiving them to be stopped, or are simply discarded if the process has

     6     SIGABRT          create core image    abort program (formerly SIGIOT)

    需要了解的信号： * 1 SIGHUP：关闭终端但是终端中仍有程序在运行，发送这个信号告诉所有流程应该关闭，类似于命令行中通讯中的挂断 * 2 SIGINT：中断程序执行 * 3 SIGQUIT：退出程序的执行，在大多数情况下和 2 SIGINT 功能相同 * 15 SIGTERM：在大多数情况下功能和 SIGINT、SIGQUIT 相同 * 17 SIGSTOP：暂停程序的执行，然后可以使用 19 SIGCONT 继续执行程序 * 19 SIGCONT：继续执行程序

    示例 signal.py 

    #!/usr/bin/env python
import signal, time

def handler (signum, time):
    print("\nI got a SIGINT, but I am not stopping")

signal.signal(signal.SIGINT, handler)
i = 0
while True:
    time.sleep(.1)
    print("\r{}".format(i), end="")
    i += 1

    运行此 Python 程序 

    python sigint.py
    效果如下（按下 Ctrl-C 发送 SIGINT 信号，无法终止，因为编写了对这个信号进行处理的函数；按下 Ctrl-\ 发送 SIGQUIT 信号，程序终止）

    注：

    • 程序无法捕获 SIGKILL 信号
    • 如果希望关闭进程也能继续工作，则可以编写装饰器以忽略信号

    username@ubuntu:~$ sleep 1000
^Z

    1.2 特殊符号

    nohup 表示 bi hang up（不挂起），在系统后台不挂断地运行命令，退出终端不会影响程序的运行；通过 & 令程序在后台运行，即，不会在程序运行时无法执行其他命令 

    username@ubuntu:~$ nohup sleep 1000 &
[2] 51432
username@ubuntu:~$ nohup: ignoring input and appending output to 'nohup.out'
^C
username@ubuntu:~$ jobs
[1]+  Stopped                 sleep 1000
[2]-  Running                 nohup sleep 1000 &
username@ubuntu:~$ sleep 1000 &
[3] 52509

    注：我这里显示的是 Stopped，而课程中显示的是 suspended

    发现问题：

    • 执行 nohup sleep 1000 & 后，无法键入其他命令
    • 关闭 terminal，重新开启一个后，jobs 和 ps -au 均未找到之前执行的命令

    注：我这里得到的和课程中的不一致，教程中的如下 

    ~/cmd >>> nohup sleep 2000
[2] 63567
appending output to nohup.out
~/cmd >>>
    此时可以键入其他命令

    通过 bg 重新执行被挂起的任务（我理解是继续执行），通过 %job编号 指定 job

    执行 bg 后的程序，会自动切换为后台运行

    username@ubuntu:~/Desktop$ sleep 1000
^Z
[1]+  Stopped                 sleep 1000
username@ubuntu:~/Desktop$ jobs
[1]+  Stopped                 sleep 1000
username@ubuntu:~/Desktop$ bg %1
[1]+ sleep 1000 &
username@ubuntu:~/Desktop$ jobs
[1]+  Running                 sleep 1000 &

    如果 bg 不指定 job，则继续执行最后一个 

    username@ubuntu:~/Desktop$ sleep 1000
^Z
[1]+  Stopped                 sleep 1000
username@ubuntu:~/Desktop$ sleep 2000
^Z
[2]+  Stopped                 sleep 2000
username@ubuntu:~/Desktop$ jobs
[1]-  Stopped                 sleep 1000
[2]+  Stopped                 sleep 2000
username@ubuntu:~/Desktop$ bg
[2]+ sleep 2000 &
username@ubuntu:~/Desktop$ jobs
[1]+  Stopped                 sleep 1000
[2]-  Running                 sleep 2000 &

    bg 可以指定多个 job 

    username@ubuntu:~/Desktop$ kill -STOP %2
username@ubuntu:~/Desktop$ jobs
[1]-  Stopped                 sleep 1000
[2]+  Stopped                 sleep 2000
username@ubuntu:~/Desktop$ bg %1 %2
[1]- sleep 1000 &
[2]+ sleep 2000 &
username@ubuntu:~/Desktop$ jobs
[1]-  Running                 sleep 1000 &
[2]+  Running                 sleep 2000 &

    通过 kill 命令结束 job 

    username@ubuntu:~/Desktop$ kill %1
username@ubuntu:~/Desktop$ jobs
[1]+  Terminated              sleep 1000

    kill 命令可以发送任何信号 

    username@ubuntu:~/Desktop$ sleep 1000
^Z
[1]+  Stopped                 sleep 1000
username@ubuntu:~/Desktop$ jobs
[1]+  Stopped                 sleep 1000
username@ubuntu:~/Desktop$ bg %1
[1]+ sleep 1000 &
username@ubuntu:~/Desktop$ jobs
[1]+  Running                 sleep 1000 &
username@ubuntu:~/Desktop$ kill -STOP %1
username@ubuntu:~/Desktop$ jobs
[1]+  Stopped                 sleep 1000

    kill 命令可以指定多个 job 

    username@ubuntu:~/Desktop$ jobs
[1]-  Running                 sleep 1000 &
[2]+  Running                 sleep 2000 &
username@ubuntu:~/Desktop$ kill -STOP %1 %2
username@ubuntu:~/Desktop$ jobs
[1]+  Stopped                 sleep 1000

    2 终端多路复用器（Terminal Multiplexers）

    为了同时在 terminal 进行多项工作，我们可能经常会打开多个 terminal 窗口，但是终端多路复用器可以更方便地达到这个目的。

    使用 tmux 可以创建不同的工作空间。也可以使用一些比较旧的工具，如 screen。

    2.1 tmux 中的 3 个核心概念

    • sessions
    • windows：类似于其他编辑器或者浏览器中的标签
    • panes

    三者的关系：panes in windows, windows in sessions

    2.2 tmux 的基本使用

    • 通过 tmux 开启一个 session，看上去与之前的 shell 没什么区别，但是这个新开启的 session 与之前的 shell 进程是分开的
    • 通过 tmux new -s session名称 在创建 session 时指定 session 的名称
    • 通过 tmux new -s session名称 -t group名称 在创建 session 时指定 session 的名称和这个 session 所属的 group 的名称
    • 通过 Ctrl-B-D 退出 session（先按 Ctrl-B，这两个键都松开，再按 D），也可以通过键入 tmux detach 退出 session
    • 通过 tmux a 回到最近退出的且还存在的 session
    • 通过 tmux a -t session名称 指定要回到的 session 的名称
    • 通过 Ctrl-D 关闭当前 session
    • 在 session 中，通过 Ctrl-B-: 进入命令行模式（先按 Ctrl-B，这两个键都松开，再按 `Shift-;）
    • 在 session 中通过 Ctrl-B-C 创建并打开一个新的 window
    • 在 window 中通过 Ctrl-B-P 进入上一个 window（P 代表 previous）
    • 在 window 中通过 Ctrl-B-N 进入下一个 window（N 代表 next）
    • 在 window 中通过 Ctrl-B-编号 进入下一个 指定编号的 window（window 编号从 0 开始）
    • 在 window 中通过 Ctrl-B-, 重命名 window
    • 通过 Ctrl-B-" 将当前 window 在水平方向上分隔，在当前 terminal 再打开一个界面（pane）
    • 通过 Ctrl-B-% 将当前 window 在竖直方向上分隔，在当前 terminal 再打开一个界面（pane）
    • 通过 Ctrl-B-space 将当前的 panes 水平等分排列，之后再次按 Ctrl-B-space 可以切换多种布局方式
    • 通过 Ctrl-B-方向键 切换所在的 pane
    • 通过 Ctrl-B-Z 令当前的 pane 填充整个 terminal，再次按下 Ctrl-B-Z 恢复到多个 pane
    • 在 window 中通过 Ctrl-B-W 打开当前 session 中的 window 列表，通过上下方向键和 ENTER 键选择（切换）window，通过左右方向键可以打开 window 的 pane
    • （我发现）在同一个 group 中的不同的 session 拥有相同的 windows

    3 通过“点文件”（Dotfiles）配置环境

    3.1 查看现有别名

    通过 alias 查看现有别名 

    username@ubuntu:~/Desktop$ alias
alias alert='notify-send --urgency=low -i "$([ $? = 0 ] && echo terminal || echo error)" "$(history|tail -n1|sed -e '\''s/^\s*[0-9]\+\s*//;s/[;&|]\s*alert$//'\'')"'
alias egrep='egrep --color=auto'
alias fgrep='fgrep --color=auto'
alias grep='grep --color=auto'
alias l='ls -CF'
alias la='ls -A'
alias ll='ls -alF'
alias ls='ls --color=auto'

    也可以为经常写错的命令创建别名，例如 

    username@ubuntu:~$ alias sl=ls
    注：根据个人习惯决定是否这样处理，因为错误也是一种有用的反馈。此外，这种处理也可能会引发一些问题，例如，将本来就存在的命令或程序覆盖。

    可以用于省略参数，例如 

    username@ubuntu:~$ alias mv="mv -i"

    通过 alias 命令 查看这个命令是什么命令的别名 

    username@ubuntu:~$ alias ll
alias ll='ls -lah'

    3.3 永久修改别名

    bash 的配置保存在当前用户 home 的 .bashrc 文件中

    查看 当前用户的 .bashrc 文件，可以在其中添加别名，例如 

    alias sl=ls
    通过 source ~/.bashrc 使其生效 
    username@ubuntu:~/Desktop$ source ~/.bashrc
username@ubuntu:~/Desktop$ sl
 nohup.out  'Parallels Shared Folders'

    也可以通过执行 bash 命令，或者重新打开 terminal 使其生效

    3.4 修改 bash 提示字符串

    通过 PS1="要定义的字符串格式" 

    username@ubuntu:~$ PS1="> "
>


    username@ubuntu:~/Desktop$ PS1="\w >"
~/Desktop >

    同样可以将这些内容写在 ~/.bashrc 中使其永久生效
